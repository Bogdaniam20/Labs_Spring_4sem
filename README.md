## Описание

В данной лабораторной работе проведена реализация и проверка API с использованием **Swagger UI**. Основное внимание было уделено обработке ошибок и логированию действий. Работа включает в себя несколько ключевых улучшений, которые значительно повышают качество приложения.

## Изменения и улучшения

1. **Обработка ошибок:** 
   - Добавлена обработка **400-х** и **500-х** ошибок, что позволяет более корректно реагировать на неверные запросы и внутренние ошибки сервера.
2. **Логирование:** 
   - Реализовано логирование действий и ошибок с использованием аспектно-ориентированного программирования (AOP), что обеспечивает более эффективный мониторинг работы приложения.
3. **Swagger:** 
   - Добавлен **Swagger UI** для удобного тестирования API.
   - *Потрачено 4 часа на интеграцию из-за несовместимости версий😔😔😔💢💢💢.*
4. **CheckStyle:** 
   - Подключен **CheckStyle** для проверки кодирования.
   - Использован собственный XML файл для конфигурации, что позволило убрать стилистические ошибки.

## Использование

Для проверки API используйте **Swagger UI**, который предоставляет интерфейс для взаимодействия с приложением. Вы сможете протестировать все доступные эндпоинты и убедиться в корректности их работы.

## Задание

В рамках лабораторной работы были выполнены следующие задачи:

1. Обработаны **400** и **500** ошибки.
2. Добавлено логирование действий и ошибок с использованием аспектов.
3. Подключены **Swagger** и **CheckStyle**. Убраны стилистические ошибки.
