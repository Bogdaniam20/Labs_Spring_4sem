## Описание

В данной лабораторной работе реализован POST метод для работы со списком параметров, передаваемых в теле запроса, с целью выполнения bulk операций. Основное внимание уделено использованию Java 8, включая Stream API и лямбда-выражения, а также обеспечению высокого уровня покрытия юнит-тестами.

## Изменения и улучшения

1. **POST метод для bulk операций:** 
   - Реализован POST метод, который принимает список параметров в теле запроса. Использованы возможности Java 8, такие как Stream API и лямбда-выражения, для обработки данных, что улучшает читаемость и производительность кода.

2. **Покрытие Unit-тестами:** 
   - Проведено покрытие бизнес-логики юнит-тестами на более чем 80%. Это обеспечивает надежность функционала и позволяет быстро выявлять и исправлять ошибки.

3. **Обработка ошибок:** 
   - Добавлена обработка ошибок для улучшения реакции на неверные запросы и внутренние ошибки сервера.

4. **Логирование:** 
   - Реализовано логирование действий и ошибок, что помогает отслеживать работу приложения и упрощает процесс отладки.

5. **Swagger:** 
   - Интегрирован **Swagger UI** для удобного тестирования и документирования API.

## Использование

Для проверки API используйте **Swagger UI**, который предоставляет интерфейс для взаимодействия с приложением. Вы сможете протестировать новый POST метод для bulk операций и убедиться в корректности его работы.

## Задание

В рамках лабораторной работы были выполнены следующие задачи:

1. Реализован POST метод для работы со списком параметров для bulk операций с использованием Java 8 (Stream API, лямбда-выражения).
2. Обеспечено покрытие юнит-тестами на более чем 80%.
3. Добавлена обработка ошибок и логирование действий.
4. Интегрирован **Swagger UI** для тестирования API.
